Definitions

"Software where LLM is in charge of the control logic"

  1. Building effective agents (Anthropic, Dec-2024)
    • "Workflows are systems where LLMs and tools are orchestrated through predefined code paths"
    • "Agents are systems where LLMs dynamically direct their own processes and tool usage"
    • "Workflows & Agents = Agentic systems"

"Software where LLM is in charge of some of the control logic"

  1. Introduction to Operator & Agents (OpenAI, Jan-2025)
    • "AI agents are AI systems that can do work for you independently; you just give them a task"
  2. What are AI agents? (IBM, Jul-2024)
    • "AI agents are compound AI systems where an LLM is in charge of the control logic"
  3. Introduction to Agents (Hugging Face, Jan-2025)
    • "AI agents are programs where LLM outputs control the workflow"
    • "Spectrum as you give more or less power to the LLM on your workflow"
  4. Agents Course (Hugging Face, Feb-2025)
    • "An AI model capable of reasoning, planning, and interacting with its environment"
    • "An Agent is a system that leverages an AI model to interact with its environment in order to achieve a user-defined objective"
  5. What is an AI agent? (LangChain, Jun-2024)
    • "An AI agent is a system that uses an LLM to decide the control flow of an application"

Classifications

  1. By how they are BUILT: Code (or Framework) vs. Assisted Code (Chat) vs. No-code (UI)
  2. By APPROACH: Graph (agent controls some of the logic) vs. Objective (agent controls all of the logic)
    • Not all graphs are agents; in agents LLM must control at least some of the logic
  3. By how they are USED: Code (or API) vs. UI
  4. By how they REASON: in Language vs. in Code (e.g. Smolagents CodeAgent)
  5. By how they are TRIGGERED: Reactive (when called) vs. Proactive (e.g. LangGraph Ambient Agents)
  6. By their AMBITION: Automate something we were already doing vs. Explore doing something new
  7. By their MARKET: B2B (Enterprise; SMB) vs. B2C (Consumer)
  8. By their TARGET: Vertical (one function e.g. Cursor) vs. Horizontal (across functions e.g. Ottogrid)
    • Open to debate where to draw the line
  9. By INTERACTION MODEL: One-off vs. Iterative (Chat)
  10. By how they are SOLD: Ad-hoc automation vs. Productized agent
